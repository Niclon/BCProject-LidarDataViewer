Controls
Short Cut Funcionality
'ctrl' Stop scene and invoke drawing to canvas on top of Three.js scene.
'shift' Accept drawn selection and create selection to the Three.js scene.
'alt' Throw away drawn selections and resume scene.
'ctrl + alt + i' Open Aframe inspector window with orthographic camera.
'M' Enable moving of selections.
'del' Delete selection which my cursor point to. Works only if moving is enabled ('M') and scene is not stopped('ctrl').
Structure of frontend
Javascript file Funcionality
'initialize.js' File for initialization of preact component(scene, camera) + selection funcionality(making selection border inside scene).
'initializeLoad.js' File for initialization of components for replay funcionality.
'lidarPoints.js' Preact component for getting points from server, rendering points into scene, changing spherical picture, taking pictures of selections and sending them to server, creating custom frustum to get all point inside selections and sending them to server too. Plus funcionality for replay. Initialization of component count with parameters 'stepNumber' current frame, 'maxStepNumber' maximal number of frames(this is here for intialization of slider GUI for frame selection) and 'isReplay' if it is replay or not;
'main.js' Preact component which creates scene camera light.
'customDrawing.js' Component which create its own canvas for drawing in real time without projecting drawing into scene. This component have its own events.
'customFrustum.js' File for storing 4 planes and having public method if point is contained in frustum. Used for deciding if point should be returned on backend or not.
'customDragControls.js' File for dragging selections around main camera using method of ray tracing to get selection.
'customSelection.js' File for creating selections and creating instances of customDragControls.
